Entity Framework Namespace Collisions when working with Multiple Contexts

25 May 2015 | Entity Framework | SQL Server

I ran into the following issue the other day while integrating Entity Framework 6. It turns out to be a design limitation of the framework. Fortunately the workaround for it is fairly simple.

Reproducing the Problem

The problem can be reproduced using the following steps:

  1. Create two databases and then a new table with the same name on both of them.
  2. Next, create a new class library to host the database contexts.
  3. Now create two contexts, one for each database. Ensure that each context is created under its own namespace.
  4. The moment we attempt to interact with the context, we get the exception.

The Solution

The problem appears to be how Entity Framework handles namespaces and their resolution, this post goes into more detail. The solution itself is pretty simple and is more of inconvenience than a real problem.
